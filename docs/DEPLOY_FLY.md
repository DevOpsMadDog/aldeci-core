# ALDECI — Fly.io Deployment Guide

> Audience: DevOps / founder doing first production deploy
> Estimated time: ~20 min (excluding DNS propagation)
> Fly.io region recommended: `iad` (US-East) or nearest to customer

---

## Step 0 — Prerequisites

```bash
# Install flyctl (macOS)
brew install flyctl

# Or via the official install script (Linux / CI)
curl -L https://fly.io/install.sh | sh

# Authenticate (opens browser)
flyctl auth login

# Verify
flyctl version   # must be >= 0.2.0
```

**Payment method**: Fly.io requires a credit card on file even for apps within the free tier.
Add one at https://fly.io/dashboard/personal/billing before running `flyctl launch`.

---

## Step 1 — Launch (no deploy yet)

Run from the repo root. Fly reads `Dockerfile` by default.

```bash
cd /path/to/Fixops

flyctl launch \
  --name aldeci \
  --region iad \
  --no-deploy \
  --dockerfile Dockerfile
```

This writes `fly.toml` to the repo root. Commit it:

```bash
git add fly.toml
git commit -m "chore(fly): add fly.toml generated by flyctl launch"
```

**Tip**: If you want to use the demo image instead: `--dockerfile Dockerfile.demo`.

## Step 2 — Set Secrets

Set all sensitive values via `flyctl secrets` — they become env vars inside the container at runtime. Never commit these to git.

```bash
flyctl secrets set \
  FIXOPS_API_TOKEN=<generate-with-openssl-rand-hex-32> \
  FIXOPS_JWT_SECRET=<generate-with-openssl-rand-hex-64> \
  OPENROUTER_API_KEY=<your-openrouter-key> \
  FIXOPS_SMTP_HOST=smtp.sendgrid.net \
  FIXOPS_SMTP_PORT=587 \
  FIXOPS_SMTP_USER=apikey \
  FIXOPS_SMTP_PASSWORD=<sendgrid-api-key> \
  FIXOPS_SMTP_FROM=noreply@aldeci.example.com
```

Verify secrets are registered (values are never shown):

```bash
flyctl secrets list
```

## Step 3 — Persistent Volume

SQLite databases and the AgentDB (`.swarm/memory.db`) must survive deploys. Create a 10 GB volume:

```bash
flyctl volumes create fixops_data --size 10 --region iad
```

Then add the mount to `fly.toml` under the `[mounts]` section (add if absent):

```toml
[[mounts]]
  source      = "fixops_data"
  destination = "/app/data"
```

Ensure the app writes all SQLite files and `.swarm/` under `/app/data`. The env var `FIXOPS_DATA_DIR=/app/data` controls this in `docker-entrypoint.sh`.

---

## Step 4 — Deploy

Build happens on Fly's remote builder — no local Docker daemon required. This is critical for M-series Macs where cross-compilation is slow.

```bash
flyctl deploy --remote-only
```

Watch live build logs stream to your terminal. First deploy takes ~3–5 min (layer cache is cold). Subsequent deploys are ~60 s.

**Scale**: default is 1 shared-cpu-1x (256 MB). For production:

```bash
flyctl scale vm shared-cpu-2x --memory 512
```

---

## Step 5 — Custom Domain

Point your domain at the app after deploy completes.

```bash
# Issue a managed TLS cert (Let's Encrypt, auto-renews)
flyctl certs create aldeci.example.com
```

**DNS records** — add both at your registrar / Cloudflare:

| Type  | Name             | Value (from `flyctl ips list`) |
|-------|------------------|-------------------------------|
| A     | aldeci           | `<IPv4 from flyctl ips list>` |
| AAAA  | aldeci           | `<IPv6 from flyctl ips list>` |

```bash
# Get your IPs
flyctl ips list

# Check cert status (may take 2–5 min after DNS propagates)
flyctl certs check aldeci.example.com
```

## Step 6 — Verify

```bash
# App and VM status
flyctl status

# Tail live logs
flyctl logs

# Health check — replace with your actual domain
curl -sf https://aldeci.example.com/api/v1/health && echo "HEALTHY" || echo "UNHEALTHY"

# Or against the Fly internal URL (no custom domain needed)
curl -sf https://aldeci.fly.dev/api/v1/health && echo "HEALTHY"
```

Expected response:

```json
{"status": "healthy", "version": "...", "scanners": 8}
```

## Rollback

List all releases and redeploy a previous image by its label:

```bash
# See release history
flyctl releases list

# Roll back to a specific release (e.g., v42)
flyctl deploy --image-label v42
```

Fly keeps the last 10 images cached on the builder — rollback is instant, no rebuild needed.
